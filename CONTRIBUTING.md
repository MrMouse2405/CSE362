## Architecture Overview

This repository is split into two main development environments that are merged during the release process:

* **Frontend (`/frontend`):** A Svelte Single Page Application (SPA) built with Bun, Vite, and shadcn/ui.

* **Backend (`/backend`):** A REST API built with Python and FastAPI, managed by `uv`.

During production releases, the Svelte frontend is compiled into static files and served directly by the FastAPI backend.

## Development Environment

We use **Nix** to ensure a reproducible development environment. You do not need to install Python, `uv`, or `bun` globally on your system.

### Prerequisites
1. Install [Nix](https://nixos.org/download/).
2. Enable [Nix Flakes](https://nixos.wiki/wiki/Flakes#Enable_flakes).

### Getting Started

1. **Clone the repository:**
   ```bash
   git clone [https://github.com/yourusername/your-repo.git](https://github.com/yourusername/your-repo.git)
   cd your-repo

2. **Enter the Nix development shell:**

This will automatically pull in the exact versions of Python, uv, and bun required for this project.

```
nix develop
```

## Running App Locally

Because the frontend and backend act independently during development, you will need to run both development servers concurrently.

### 0. Enter nix-shell

Remember, the required packages are not installed globally, they are installed in a self-contained nix-environment. Therefore, you must enter nix shell by running:

```bash
nix develop
```

Everytime you open a new terminal.

### 1. Start the Backend (FastAPI)

The backend uses uv for dependency management.

```bash
cd backend
# uv will automatically create a virtual environment and resolve dependencies
uv run fastapi dev
```

The API is now running at http://127.0.0.1:8000

To run tests:

``` 
uv run pytest
```

### 2. Start the Frontend (Svelte + Bun)

Open a second terminal window, run `nix develop` again to enter the environment, and start the Vite dev server.

```bash
cd frontend
# Install dependencies (only needed the first time or when package.json changes)
bun install
# Start the development server
bun run dev
```
The Svelte app is now running (usually at http://localhost:5173). Vite is configured to automatically proxy `/api` requests to your local FastAPI server.

## Making Changes

1. **Create a branch**: Create a new branch for your feature or bugfix (`git checkout -b feature/my-new-feature`).

2. **Write your code**: Make your changes in either the `/frontend` or `/backend` directories.

3. **Test**: Ensure your changes do not break existing functionality.

4. **Commit**: Write clear, concise commit messages.

## Submitting a Pull Request

1. Push your branch to your fork or the main repository.

2. Open a Pull Request against the main branch.

3. Provide a clear description of the problem you are solving and the solution you implemented.

4. Once approved, a maintainer will merge your PR.

*Note: Pushing a new tag (e.g., v1.x.x) to the repository will automatically trigger the GitHub Action release pipeline, generating the static Svelte build and packaging it with the FastAPI backend into a pip-ready release.*
